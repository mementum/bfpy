Class BfApi

source code

        BfApi
Known Subclasses:

The class implements a unified communication interface with the Betfair API Services EndPoints (Global, ExchangeUK, ExchangeAus)

This is done by defining the the services and object retrieval methods by means of non-data descriptors. The descriptors are added to the class by the metaclass {BfService}

The class can be configured to avoid any pre/post-processing of the requests and answers by changing the module preProcess and postProcess variables or passing them as named arguments to the constructor

invoke(self, methodName, service, request, skipErrorCodes)

source code 

Invokes a service with a given request and with a list of errors that do not generate exceptions if returned

Parameters:
  • methodName (str) - method name that has been used to invoke
  • service (method generated by suds from the WSDL definition) - name of the object to be retrieved
  • skipErrorCodes (list) - error codes that will not generate exceptions
Returns: object generated by suds from the WSDL definitions
the response object processed by suds
Raises:
  • BfHttpError - if the server does not answer with 200 OK
  • BfNetworkError - on (potential) network errors
  • Exception - on other errors, due to the fact that suds does not generate all exceptions from a common base
  • BfApiError - on specific API errors
  • BfServiceError - on service errors (unless specified in skipErrorCodes)
