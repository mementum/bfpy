Class Hierarchy

  • logging.Filterer: A base class for loggers and handlers which allows them to share common code.
    • logging.Handler: Handler instances dispatch logging events to specific destinations.
      • bfpy.util.NullHandler: Definition of a Null logging class to avoid any output from suds
  • suds.transport.Transport: The transport interface.
    • bfpy.bftransport.BfTransport: HTTP transport using urllib2.
  • object: The most base type
    • bfpy.bfprocessors.ArrayFix: Response processor
    • bfpy.bfprocessors.ArrayUnfix: Request processor
    • exceptions.BaseException: Common base class for all exceptions
      • exceptions.Exception: Common base class for all non-exit exceptions.
        • bfpy.bferror.BfError: Base BfPy exception class.
          • bfpy.bferror.BfBetfairError: Derived from BfError.
            • bfpy.bferror.BfApiError: Derived from the Betfair Catch-all exceptions for API errors
            • bfpy.bferror.BfApplicationError: Derived from the Betfair Catch-all exceptions for application specific errors.
            • bfpy.bferror.BfServiceError: Derived from the Betfair Catch-all exceptions for Service errors
          • bfpy.bferror.BfPythonError: Derived from BfError, holding potential Python errors (like missing method or syntax errors)
          • bfpy.bferror.BfSudsError: Derived from BfError, holding potential Suds errors Not all suds errors can be caught, due to lack of unified exception generation in suds
          • bfpy.bferror.BfTransportError: Derived from BfError, to hold error generated by BfTransport
            • bfpy.bferror.BfHttpError: Derived from BfError, to hold error errors at Http level
            • bfpy.bferror.BfNetworkError: Derived from BfError, to hold error (potential) network errors
    • bfpy.bfapi.BfApi: The class implements a unified communication interface with the Betfair API Services EndPoints (Global, ExchangeUK, ExchangeAus)
      • bfpy.bfclient.BfClient: The class defines a higher level interface over the BfApi parent class.
    • bfpy.util.EmptyObject: Used to implement objects, like those arising from compressed answers
    • bfpy.bfprocessors.PatchAttr: Response processor
      • bfpy.bfprocessors.PatchAttrExId: Response processor
    • bfpy.bfprocessors.PreProcLogin: Request processor
    • bfpy.bfprocessors.PreProcMarketProfitAndLoss: Request processor
    • bfpy.bfprocessors.ProcAllMarkets: Response processor
    • bfpy.bfprocessors.ProcCurrentBets: Response processor
    • bfpy.bfprocessors.ProcLogin: Response processor
    • bfpy.bfprocessors.ProcMarket: Response processor
    • bfpy.bfprocessors.ProcMarketPricesCompressed: Response processor
    • bfpy.bfprocessors.ProcMarketProfitAndLoss: Response processor
    • bfpy.bfservice.ServiceDescriptor: Base abstract descriptor class for the implementation of the methods provided by the Betfair API
      • bfpy.bfclientext.GetCurrentBets: Implements a non-data descriptor for BfClient to extend the getCurrentBets service method by allowing betStatus to be MU (return both Matched and UnMatched bets in a single call) Betfair does not allow it, which is odd.
      • bfpy.bfclientext.GetEvents: Implements a non-data descriptor for BfClient to extend the getEvents service method by unifying getActiveEventTypes and getEvents to be able to directly recurse downwards the list of events from a fictitious eventRootId (-1)
      • bfpy.bfclientext.PlaceBets: Implements a non-data descriptor for BfClient to extend the placeBets to re-place a bet if the betPersistence has not been speficied.
      • bfpy.bfservice.ServiceDef: Definition and implementation of a service.
        • bfpy.bfservice.ExchangeServiceDef: Service Definition for exchane services.
        • bfpy.bfservice.GlobalServiceDef: Service Definition for global services.
      • bfpy.bfservice.ServiceObject: Service descriptor implementation to retrieve named objects defined for the services.
        • bfpy.bfservice.ExchangeObject: Subclass of ServiceObject to retrieve objects from the exchange endPoint (1)
        • bfpy.bfservice.GlobalObject: Subclass of ServiceObject to retrieve objects from the global endPoint (0)
    • tuple: tuple() -> empty tuple tuple(iterable) -> tuple initialized from iterable's items
      • bfpy.bfapi.BfApi.MinStakes: CurrencyMinStakes(minimumStake, minimumRangeStake, minimumBSPLayLiability)
    • type: type(object) -> the object's type type(name, bases, dict) -> a new type
      • bfpy.bfservice.BfService: Metaclass for BfApi to initialize the non-data descriptor based methods on instantiation
    • datetime.tzinfo: Abstract base class for time zone info objects.
      • bfpy.timezone.GMT: GMT (good for 0, +1 and possibly +2) Timezone.
      • bfpy.timezone.LocalTimezone: System specific local timezone.
      • bfpy.timezone.UTC: UTC Timezone class
