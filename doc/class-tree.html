Class Hierarchy

  • logging.Filterer: A base class for loggers and handlers which allows them to share common code.
    • logging.Handler: Handler instances dispatch logging events to specific destinations.
      • bfpy.util.NullHandler: Definition of a Null logging class to avoid any output from suds
  • suds.transport.Transport: The transport interface.
    • bfpy.bftransport.BfTransport: HTTP transport using urllib2.
  • object: The most base type
    • exceptions.BaseException: Common base class for all exceptions
      • exceptions.Exception: Common base class for all non-exit exceptions.
        • bfpy.bferror.BfError: Base BfPy exception class.
          • bfpy.bferror.BfBetfairError: Derived from BfError.
            • bfpy.bferror.BfApiError: Derived from the Betfair Catch-all exceptions for API errors
            • bfpy.bferror.BfApplicationError: Derived from the Betfair Catch-all exceptions for application specific errors.
            • bfpy.bferror.BfServiceError: Derived from the Betfair Catch-all exceptions for Service errors
          • bfpy.bferror.BfPythonError: Derived from BfError, holding potential Python errors (like missing method or syntax errors)
          • bfpy.bferror.BfSudsError: Derived from BfError, holding potential Suds errors Not all suds errors can be caught, due to lack of unified exception generation in suds
          • bfpy.bferror.BfTransportError: Derived from BfError, to hold error generated by BfTransport
            • bfpy.bferror.BfHttpError: Derived from BfError, to hold error errors at Http level
            • bfpy.bferror.BfNetworkError: Derived from BfError, to hold error (potential) network errors
    • bfpy.bfapi.BfApi: The BfApi is a mere "objectization" of the betfair API.
      • bfpy.betfair.Betfair: The Betfair object processes Betfair API answers (if needed) and decompresses compressed answers into usable objects
    • bfpy.util.EmptyObject: Used to implement objects, like those arising from compressed answers
    • tuple: tuple() -> empty tuple tuple(iterable) -> tuple initialized from iterable's items
      • bfpy.bfapi.MarketTuple: MarketTuple(exchangeId, marketId)
    • datetime.tzinfo: Abstract base class for time zone info objects.
      • bfpy.timezone.GMT: GMT (good for 0, +1 and possibly +2) Timezone.
      • bfpy.timezone.LocalTimezone: System specific local timezone.
      • bfpy.timezone.UTC: UTC Timezone class
