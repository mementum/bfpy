GetActiveEventTypes(self)

source code 

Retrieve the list of Active Event Types (top level events)

Removes one level of idirection for eventTypeItems

It performs an "eventItemization" of the top level events.

The reasoning behind this is to let the UI manage only two types of objects: -- Events (eventItems) -- Markets (marketItems)

EventTypes have an exchangeId field for Greyhound races and Events have fields like menuLevel and orderIndex, but the main fields are actually the same:

-- id vs eventId -- name vs eventName

Returns: suds object
Betfair API answer
Overrides: bfapi.BfApi.GetActiveEventTypes

source code 

Get Matched and Unmatched bets for a given marketId

Removal of indirection for bets is performed

Parameters:
  • exchangeId (int) - id of the suds "exchange" wsdl client to use
  • marketId (int) - id of the market
  • betStatus (string (Betfair enum)) - status of bets to be returned (matched, unmatched)
  • orderBy (string (Betfair enum)) - ordering criterion
  • sortOrder (string (Betfair enum)) - ordering criterion (asc/desc)
  • startRecord (int) - to enable paging through long lists
  • recordCount (int) - maximum number of records to retrieve (see matchedSince)
  • excludeLastSecond (bool) - exclude changes in the last second
  • useMatchedSince (bool) - whether to use the matched since param
  • matchedSince (datetime) - date to return bets since. It should void recordCounts
Returns: suds object
Betfair API answer
