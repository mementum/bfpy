Class BfClient

source code

 object --+    
          |    
bfapi.BfApi --+

             BfClient

The class defines a higher level interface over the BfApi parent class.

The intention is to have some services redefine/extend the functionality in a intelligent manner.

As an example de redefined "getEvents" unifies getActiveEventTypes and getEvents (it is used as such in the Bfplusplus application <http://code.google.com/p/bfplusplus>

The redefintion is made by means of non-data descriptors in the bfclientext module

placeBets(...)

source code 

Implements a non-data descriptor for BfClient to extend the placeBets to re-place a bet if the betPersistence has not been speficied.

This is a hack to support Bfplusplus, given that the "marketSummary" returned by getEvents does not state if a market will turn in-ply or not. A call to getAllMarkets is needed to find out.

This hack will be removed

Overrides: bfapi.BfApi.placeBets
